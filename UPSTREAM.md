# আপস্ট্রিম (Summit / BTCL) অ্যাপের সাথে কীভাবে যুক্ত

## নেটওয়ার্ক লেভেলে কী হয়

1. **BTCL** আপনার কাছে একটা **ফিজিক্যাল/নেটওয়ার্ক লাইন** দেয় (ফাইবার বা E1 ইত্যাদি)।
2. সেই লাইন আপনার **রাউটার/সার্ভার** (যেখানে MikroTik চলে) এ কানেক্ট থাকে।
3. **অ্যাপ BTCL এর নেটওয়ার্কে সরাসরি কানেক্ট হয় না** – BTCL সাধারণত ISP অ্যাপের জন্য কোনো API দেয় না।  
   মানে: অ্যাপ দিয়ে BTCL লাইন “অন/অফ” বা “স্পিড চেঞ্জ” করা যায় না।

## অ্যাপ কী করে

অ্যাপ কাজ করে **আপনার নেট** নিয়ে – অর্থাৎ যে লাইন BTCL দিয়েছে, সেই লাইন দিয়ে আপনি রিসেলার/কাস্টমারদের নেট দেন, সেটাই অ্যাপ ম্যানেজ করে:

| জিনিস | অ্যাপে কোথায় |
|--------|-----------------|
| কে কত স্পিড পাবে | **Packages** + **MikroTik Sync** (রাউটারে লিমিট সেট হয়) |
| কাস্টমার/রিসেলার বিল | **Bills**, **Reports**, **Collection** |
| আপস্ট্রিম বিল (Summit/BTCL) | **Reports → Add Expense → UPSTREAM** (মাসিক খরচ লিখে রাখা) |
| আপস্ট্রিম থেকে কত Mbps নিয়েছেন | **Upstream** সেটিং – প্রোভাইডার (যেমন Summit) ও ক্যাপাসিটি লিখে রাখা, অ্যাপ দেখাবে কত Mbps “সেল” করা আছে |

## সংক্ষেপে কানেকশন

```
আপস্ট্রিম (Summit / BTCL) → আপনার রাউটার/সার্ভার (MikroTik) → রিসেলার/কাস্টমার
                     ↑
                অ্যাপ শুধু এখানকার অংশ ম্যানেজ করে:
                – কে কত স্পিড পাবে (MikroTik sync)
                – বিল, কালেকশন, রিপোর্ট
                – BTCL বিল/ক্যাপাসিটি রেকর্ড (Reports + Upstream সেটিং)
```

অর্থাৎ **Summit বা অন্য প্রোভাইডার থেকে যে লাইন নেবেন সেটা ফিজিক্যালি রাউটারে কানেক্ট থাকবে; অ্যাপে সেটা “কানেক্ট” করার মানে হলো – অ্যাপে **আপস্ট্রিমের তথ্য (প্রোভাইডার, ক্যাপাসিটি, খরচ) রেকর্ড রাখা** এবং **কত স্পিড সেল করা আছে সেটা দেখা**।
